# AWS CodeDeploy

## Overview

AWS CodeDeploy is a fully managed deployment service that automates application deployments to various compute services such as Amazon EC2, AWS Lambda, Amazon ECS, and on-premises servers.

In a CI/CD pipeline, CodeDeploy is responsible for delivering the built application to the target environment, ensuring consistent and reliable deployments.

It reduces the need for manual deployments and minimizes the risk of deployment-related errors.

---

## Why Do We Need CodeDeploy?

Deploying applications manually can lead to:

- Human errors
- Inconsistent deployments
- Downtime during releases
- Difficult rollback procedures
- Slower release cycles

CodeDeploy automates the deployment process and provides mechanisms for safer releases.

---

## Role of CodeDeploy in CI/CD

CodeDeploy serves as the **Deployment Stage** of the CI/CD pipeline.

```text
Developer
    │
git push
    │
    ▼
CodeCommit
(Source)
    │
    ▼
CodePipeline
(Orchestration)
    │
    ▼
CodeBuild
(Build & Test)
    │
    ▼
CodeDeploy
(Deployment)
    │
    ▼
Production
```

---

## What Happens Inside CodeDeploy?

### Step 1: Receive Build Artifacts

CodeDeploy receives the deployment package generated by CodeBuild.

Examples:

- ZIP files
- JAR files
- WAR files
- Application packages

### Step 2: Identify Deployment Target

CodeDeploy determines where the application should be deployed.

Supported targets include:

- Amazon EC2
- Amazon ECS
- AWS Lambda
- On-premises servers

### Step 3: Execute Deployment

The deployment process begins according to the configured deployment strategy.

### Step 4: Validate Deployment

Health checks and validation tests are performed to verify successful deployment.

### Step 5: Complete Deployment

The new application version becomes available to users.

## Deployment Strategies

### In-Place Deployment

The application is updated directly on existing servers.

```text
Old Version
     │
     ▼
Server Updated
     │
     ▼
New Version
```

**Advantages**

- Simple to configure
- Lower infrastructure cost

**Disadvantages**

- Potential downtime
- Rollback can be more challenging

---

### Blue/Green Deployment

A new environment is created alongside the existing one.

```text
Blue Environment (Current)
            │
            ▼
Green Environment (New)
            │
            ▼
Traffic Shift
```

**Advantages**

- Minimal downtime
- Safer deployments
- Easy rollback

**Disadvantages**

- Requires additional resources

## Deployment Lifecycle Hooks

CodeDeploy supports lifecycle events that allow custom scripts to run during deployment.

Common hooks include:

### BeforeInstall

Runs before application files are copied.

### AfterInstall

Runs after files are copied successfully.

### ApplicationStart

Starts the application after deployment.

### ValidateService

Verifies that the application is functioning correctly.

## Key Takeaways

- CodeDeploy is AWS's fully managed deployment service.
- It serves as the Deployment Stage in a CI/CD pipeline.
- It deploys applications to EC2, ECS, Lambda, and on-premises servers.
- It supports both In-Place and Blue/Green deployment strategies.
- It uses an `appspec.yml` file to define deployment instructions.
- It integrates seamlessly with CodePipeline and CodeBuild.
- It helps automate reliable and repeatable application deployments.
